# 文档本地化政策

DebianForm 所有维护中的 Markdown 文档都必须同时提供英文和简体中文版本。唯一例外是
`AGENTS.md`，因为它是仓库操作元数据，不属于产品、用户或维护者文档。

## 文件命名

- 仓库根目录使用 `<name>.md` 作为英文文件名，使用 `<name>.zh-CN.md` 作为简体中文文件名。
- 仓库根目录以外使用 `<name>.md` 作为英文文件名，使用 `<name>.zh.md` 作为简体中文文件名。
- 一对文件放在同一目录中，使相对资源和相邻文档使用相同的基础目录。

## 翻译要求

- 翻译必须覆盖源文档的完整内容，不能用摘要代替任何章节。
- 命令、代码、URL、哈希、版本、支持等级、标识符和验证证据必须保留原有技术含义。代码块中供人
  阅读的注释或示例文案可以本地化。
- 一对文档中的标题、围栏代码块、表格和清单必须保持等价结构。
- 两种语言中的产品行为和兼容性声明必须一致。
- 共享行为、支持状态或发布政策发生变化时，必须在同一个变更中更新两种语言。

## 导航

- 每份文档都必须在顶部附近提供醒目的语言选择器，并直接链接到对应版本。
- 英文文档默认链接到英文文档，简体中文文档默认链接到简体中文文档。
- 语言选择器是常规情况下唯一允许的跨语言链接。
- 仓库内文件优先使用相对链接。

## 验证

提交前运行文档门禁：

```bash
make docs-check
```

门禁会验证文档配对、语言选择器、结构一致性、英文正文、同语言导航和本地链接目标。CI 会执行
同一个命令。
